## Usage for Business Analyst & Product Owner

This conceptual view allows a BA/PO to:

- Differentiate between **authorisation definition** (master data of security) and **operational assignment**.
- Model **templates** as reusable permission sets that can be attached to multiple profiles.
- Define policies for **profile combination, inheritance, and conflict resolution**.
- Specify how **external IdP authentication** flows into the token + profile data merge, resulting in a **permission graph** (per system, menu, option, action, role, etc.).

Use this document as the **AGENT_CONTEXT** when generating formal requirements, use cases, data models, and test scenarios with Bmad.
